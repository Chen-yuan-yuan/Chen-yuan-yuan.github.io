第一章 绪论

操作系统的概念 *

  1. 管理和控制计算机系统中的硬件和软件资源，合理地组织计算机工作六次，方便用户使用程序集合。

操作系统的历史

计算机发展四个阶段

  • 第一阶段：电子管时代，无操作系统时代。
  • 第二阶段：晶体管时代，批处理操作系统。
  • 第三阶段：集成电路时代，多道程序设计。
  • 第四阶段：超大和大规模集成电路时代，分时操作系统。
  1. 手工操作阶段
    • 电子管时代，无操作系统时代
  2. 早期批处理
    • 联机批处理（串行）
      1. 需要作业说明书
    • 脱机批处理（并行）

通道：是一种专用处理部件，能控制一台或多台输入输出设备工作，负责输入输出设备与主存之间的信息传输。

  1. 多道程序系统
    • 多道 计算机内存中同上存放几道相互独立的程序。
    • 宏观上并行 同时进入系统的几道程序都处于运行过程中，即它们先后开始了各自的运行，但都未运行。
    • 微观上串行 实际上，各道程序轮流使用CPU，交替执行。
  2. 分时操作系统
    • 分时就是把时间分成很短的时间片，轮流处理作业
  3. 实时操作系统
    • 实时操作就是在允许的范围之内做成响应
  4. 通用操作系统
    • 它可以同时兼顾有多道批处理、分时、实时处理的功能，或其中两个以上的功能
  5. 操作系统的进一步发展
    • PC的操作系统，例如window
    • 嵌入式操作系统
    • 网络操作系统
    • 分布式操作系统
    • 智能化操作系统

  4. 通用操作系统

  5. 个人计算机的操作系统

第三章 进程管理

进程的概念 *

  1. 程序的并发执行
    • 程序：具有独立功能的，并在时间上按严格次序前后相继的计算机操作序列集合，是一个静态的概念。
    • 程序的顺序执行
      • 顺序性
      • 封闭性
      • 可再现性
    • 多道程序系统中程序执行环境的变化
      • 独立性
      • 随机性
      • 资源共享性
    • 程序的并发执行
      • 在执行时间在客观上相互重叠，即一个程序段没结束，另一个程序段已经开始执行
  2. 进程的定义
    • 并发执行的程序在执行过程中分配和管理资源的基本单位
    • 进程和程序的区别
      • 进程是一个动态概念，程序是动态概念，进程是程序的执行过程
      • 进程具有并发性，程序没有
      • 进程是竞争计算机系统资源的基本单位
      • 不同的进程可以包含同一程序

进程的描述 *

进程是一个程序对某个数据集的执行过程，是分配资源的基本单位。

  1. 进程控制块（PCB）
    • 描述信息
      • 进程名或进程标识号
      • 用户名或用户标识号
      • 家族关系
    • 控制信息
      • 进程状态
        • 就绪
        • 执行
        • 等待
        • 完成
        • 提交
    • 资源管理信息
    • 现场保护结构
  2. 进程上下文
    • 实际上是进程执行过程中顺序关联的静态描述
    • 已执行过的进程指令和数据在相关寄存器和堆栈中的内容称为上文
    • 正在执行的称为正文
    • 等待执行的称为下文
  3. 进程上下文切换
    • 进程切换原因
      • 时钟中断
      • I/O中断
      • 内容访问出错
      • 陷阱
  4. 进程空间与大小
    • 任意一进程都有自己的地址空间，称为进程空间或虚空间。

进程互斥 *

进程之间的资源竞争和共享

  1. 资源共享所引起的制约
    • 临界区：把不允许多个并发进程交叉执行的一段程序称为临界区。
    • 间接制约：临界区不允许并发进程交叉执行的现象，由共享共有资源造成的对并发进程执行速度的间接制约。
    • 什么是互斥：不允许两个以上的共享资源的并发进程同时进入临界区称为互斥
  2. 互斥的加锁实现
    • 什么是互斥的加锁：就是一个进程进入临界区，然后不允许其他进程再进入
  3. 信号量和P、V原语
    • 信号量：信号量>=0表示可供并发进程使用的资源实体数
    • P、V原语：P代表使用，使用一次信号量-1，使用一次V，信号量+1
  4. 用P、V原语实现进程互斥
    • 使用P、V原语，可以解决加锁带来的各种问题

进程同步 *

  1. 同步的概念
    • 把异步环境的一组并发进程因直接制约而互相发送消息而进行互相合作、互相等待，使得各进程按一定的速度执行的过程称为进程间的同步。
  2. 私用信号量
    • 信号量只与制约进程及被制约进程有关而不是与整组并发进程有关
  3. 用P、V原语操作实现同步
    1. 为各并发进程设置私用信号量
    2. 为私用信号量赋值
    3. 规定各进程的执行顺序
  4. 生产者-消费者问题
    • 其实就是一个同步问题

进程通信 *

  1. 进程的通信方式
    • 主从式
      • 可自由使用进程的资源和数据
      • 进程的动作受主进程的控制
      • 主进程和从进程的关系式固定的
    • 会话式
      • 使用进程在使用服务进程所提供的服务之前，必须得到服务进程的许可
      • 服务进程根据使用进程的要求提供服务，但对所提供服务的控制由服务进程自身完成
      • 使用进程和服务进程在进行通信时由固定连接关系
    • 消息或邮箱机制
      • 只要存在空缓冲区或邮箱，发送进程就可以发送消息
      • 与会话系统不同，发送进程和接受进程之间无直接连接关系，接收进程可能在收到某个发送进程发来的消息之后，又去接收另一个发送进程发来的信息
      • 发送进程和接收进程之间存在缓冲区或者邮件用来存放被传送消息1
    • 共享存储区方式
  2. 消息缓冲机制
    • 发送进程在发送消息前，先在自己的内存空间设置一个发送区
  3. 邮件通信
    • 就是由发送进程申请建立一个与接收进程链接的邮箱
  4. 进程通信的实例-控制台的通信
    • 除用户终端，还有系统操作员控制的控制台终端
  5. 进程通信的实例-管道

    进程通信的实例之以是unix系统的管道通信。

    • 管道
      • 有名管道
      • 无名管道：为建立管道的进程及其子孙提供一条以比特流方式传送消息的通信管道

死锁问题 *

  1. 死锁概念
    • 死锁的定义：指各并发进程互相等待对方所拥有的资源，且这些并发进程在得到对方的资源之前不会释放自己所拥有的资源
    • 死锁的起因：并发进程的资源竞争
    • 产生死锁的必要条件：
      • 互斥条件
      • 不剥夺条件
      • 部分分配
      • 环路条件
  2. 死锁的消除方法

    鸵鸟策略：逃避的方式

    • 预防
      1. 打破资源的互斥和不可剥夺这两个条件
      2. 打破资源的部分分配这个死锁产生的必要条件
    • 避免（动态预防）
      1. 把进程分为多个步骤
    • 检测和恢复
      1. 当进程进行资源请求时，死锁检测算法检查并发进程组是否构成资源的请求和保持环路。

线程问题 *

  1. 为什么要引入线程
    • 提高CPU的执行效率
  2. 线程的基本概念
    • 线程是进程的一部分（注意）
  3. 线程和进程的区别
    • 同上
  4. 线程的适用范围

    不是所有计算机系统中线程都是适用的

    • 服务器中的文件管理或通信控制
    • 前后台处理
    • 异步处理

线程分类和执行 *

  1. 线程的分类
    • 用户级线程
    • 内核级线程（系统级线程）
    • 不同之处：
      • 前者的调度算法和调度过程全部由用户自行选择和确定，与操作系统内核无关
      • 用户级线程的调度算法只进行线程上下文切换而不进行处理机切换，而且线程上下文切换是在内核不参与的情况下进行的
  2. 线程的执行特性
    • 基本状态
      1. 执行
      2. 就绪
      3. 阻塞

分级调度 *

  1. 作业的状态及其转换

      • 执行
      • 运行
      • 完成
  2. 调度的层次
    • 作业调度。又叫宏观调度或高级调度
    • 交换调度。又称中级调度
    • 进程调度。又称微观调度或低级调度
    • 线程调度
  3. 作业与进程的关系
    • 作业可以看成任务，进程是完成任务的执行实体

作业调度 *

  1. 作业调度功能
    • 记录系统中各作业的状况，包括执行阶段的有关情况
    • 从后备队列中挑选初一部分作业投入执行
    • 为被选中作业做好执行前的准备工作
    • 在作业执行结束时做善后处理工作
  2. 作业调度目标与性能衡量
    • 调度目标
      • 对所有作业应该是公平合理的
      • 应使用设备由高的利用率
      • 每天执行尽可能多的作业
      • 由快的响应时间
    • 衡量批处理系统的调度算法优劣
      • 平均周转时间
      • 平均带权周转时间
    • 衡量分时和实时系统的调度算法优劣
      • 平均周转时间
      • 平均带权周转时间
      • 平均响应时间

进程调度 *

  1. 进程调度的功能
    • 记录系统中所有进程的执行情况
    • 选择占有处理机的进程
    • 进行进程上下文切换
  2. 进程调度的时机
    • 正在执行的进程执行完毕
    • 激活了等待资源的进程队列
    • 时间片被用完
  3. 进程调度性能评价
    • 进程调度性能的衡量是操作系统设计的一个重要指标
    • 进程调度性能的衡量方法
      • 定性
      • 定量

调度算法

  1. 先来先服务（FCFS，适用作业和进程）
  2. 轮转法（round robin，适用进程调度）
    • 只能用来调度分配那些可以抢占的资源
  3. 多级反馈轮转法（适用进程调用）
  4. 优先级法
    • 静态优先级
    • 动态优先级
  5. 最短作业优先级（适用作业和进程）
  6. 最高响应比优先法

实时系统调度方法

  1. 实时系统的特点
    • 有限等待时间（决定性特性）
    • 有限响应时间
    • 用户控制
    • 可靠性高
    • 系统出错处处理力强
    • 上述要求下面的能力
      • 很快的进程或线程切换速度
      • 快速的外部中断响应能力
      • 基与优先级的随时抢先式调度策略
  2. 实时调度算法的分类
    • 静态表格驱动类
    • 静态优先级驱动抢先式调度算法
    • 动态计划调度算法类
    • 尽力而为调度算法类
  3. 时限调度算法与频率单调调度算法
    • 任务就绪时间或时间到达时间
    • 开始时限
    • 完成时限
    • 处理时间

  2. 交换技术
    • 大多用在小型机和微机中，这样的系统大部分采用固定的或动态分区方式管理内存

页式管理 *

  1. 页式管理的基本原理
    • 各进程的虚拟空间被划分成若干个长度相等的页
  2. 静态页式管理

    在进程开始执行前，把该作业或进程的程序段和数据全部装入内存的各个页面中，并通过页表和硬件地址变换机构实现虚拟地址到内存物理地址的地址映射

    • 内存页面分配和回收
      1. 页表
      2. 请求表
      3. 存储页面表
    • 分配算法
    • 地址变换
  3. 动态页式管理

    动态页式管理式在静态页式管理的基础上发展起来的。分为请求页式管理和预调入页式管理。

  4. 请求页式管理中的置换算法

    置换算法在内存中没有空闲页面时被调用，它的目的是选出一个被淘汰的页面。

    • 随机淘汰算法
    • 轮转法和先进先出算法
    • 最近最久未使用页面置换算法
    • 理想型淘汰算法
  5. 存储保护
    • 越界保护
    • 存取控制保护
  6. 页式管理的优缺点
    • 优点：
      1. 不要求作业或进程的程序段和数据在内存连续存放，从而有效解决了碎片问题
      2. 动态页式管理提供了内存和外存统一管理的虚存实现方式，使用户的可以利用存储空间大大增加。提高了主存的利用率，又有利于组织多道程序执行
    • 缺点：
      1. 要求又相应的硬件支持。
      2. 增加了系统开销，例如：缺页中断处理。
      3. 请求调页的算法如选择不当，有可能产生抖动现象。
      4. 虽然消除了碎片，但是每个作业或进程的最后一页总有一部分空间得不到利用。

断式与断页式管理 *

  1. 段式管理的基本思想
    • 便于编程

    • 缺点
