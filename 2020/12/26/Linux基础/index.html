Linux系统概述(第一章)

什么是操作系统

  • 略。

操作系统的功能

  1. 存储管理
    • 内存分配
    • 地址映射
    • 内存保护
    • 内存扩充
  2. 进程和处理机管理
    • 作业和进程调度
    • 进程控制
    • 进程通信
  3. 文件管理
    • 文件存储空间的管理
    • 文件操作的一般管理
    • 目录管理
    • 文件的读写管理和存取控制
  4. 设备管理
    • 缓冲区管理
    • 设备分配
    • 设备驱动
    • 设备无关性
  5. 用户接口
    • 图形用户接口
    • 命令行接口（终端）
    • 程序接口

Linux系统的历史和现状

  • Unix家族的一员
  • 1991年诞生
  • 作者：Linus Torvalds
  • GNU 是 FSF（Free Soft Foundation 自由软件基金会）的一项开放源码工程计划
  • Linux 93 年加入 GNU奉行 GPL 版权。

Linux的现状

  1. 超级计算机 和 服务器领域
  2. 嵌入式系统
  3. 云计算系统 云计算和大数据领域采用分布式系统基础架构、称为大数据心脏的Hadoop最早就是为了在Linux平台上使用而开发的。
  4. 桌面系统

Linux系统的特点

  1. 与UNIX兼容
  2. 自由软件，源码公开
  3. 性能高，安全性强
  4. 便于定制和再开发
  5. 互操作性高
  6. 全面的多任务和真正的32位操作系统

Linux的不足

  1. 没有特定的支持厂商,不像Windows系统有微软公司专门运营开发并进行版本更新。
  2. 使用不方便,命令行方式难学难记。
  3. 图形界面不如Windows7或Windows10等产品那样靓丽舒适。
  4. 应用软件少,特别是游戏软件不如Windows丰富多彩等等。

注：Linux 的 GUI Linux 的图形界面主要有 gnome 和 KDE。无论 Gnome 还是 KDE，其基础都是 X-Window 。

Linux的常用版本

  1. 内核版本（Linux内核的官方版本由 LinusTorvalds 本人维护）
  2. 发行版本

Linux常用命令(第二章)

提示符

  • “$” 表示普通用户。
  • “#” 表示超级用户。

转义字符

  • PS1=“$ ”

    • \H　主机名。(考)

    • \u　当前用户的用户名。(考)

    • \w　当前的工作目录。(考)

    • \d　以“星期 月 日”形式表示的日期,如Mon　Dec　1。

    • \h　主机名,直至第一个“.”为止。

    • \s　所用shell名称。

    • \t　按24小时制(即小时:分:秒)形式表示的当前时间。

    • \T　按12小时制(即小时:分:秒)形式表示的当前时间。

    • @　按12小时制am/pm 形式表示的当前时间。

    • \v　bash的版本号。

    • $　如果有效的UID(用户标识)为0,那么它就是一个“#”;否则,就是一个“$”

简单命令

  • who 列出所有正在使用系统的用户、所用终端名和注册到系统的时间。
    • -b,–boot 显示最近引导时间。
    • -H,heading 打印各列的标题。
    • -m 显示与标准输入相关的主机名和用户。
    • -q,–count 显示所有注册用户名和注册用户数目。
    • -t,time 显示系统时间最后修改情况。
  • echo 打印输出
  • clear 清屏
  • password 改自己的密码
  • ls 列出指定目录的内容
    • -d 了解目录文件的使用权限
  • pwd 显示当前目录的路径。
  • su 更改用户的身份

命令格式

  • 一般命令格式：命令名 [选项] [参数] …

文件及其类型

  1. 文件。
  2. 文件的成分。
    • 索引节点
    • 数据
  3. 文件命名。
    • 最多255个字符。
    • Linux系统区分文件名的大小写。
    • 避免特殊含义的字符。例如：; | < > ˋ ″ ′ $ ! % & * ? \ ( ) [ ]
    • 以.开头的文件是隐含文件。
  4. 文件名通配符。
    • *: 0个或多个任意的字符。
    • ?: 单个任意字符。
    • [ ]: 匹配单个任意的字符匹配。
      1. -: 代表一个范围的字符
      2. ！或 ^：表示不在方括号出现的字符。

文件类型

  1. 普通文件。
    • 文本文件
    • 数据文件
    • 可执行的二进制程序
  2. 目录。
    • …表示上一级目录
    • . 表示当前目录
  3. 设备文件。
  4. 符号链接文件。

常用文件操作命令

有关文件显示命令

  1. cat 其一是经常用来显示文件的内容；其二是连接两个或多个文件。
  2. more 命令显示文件内容，每次显示一屏。
  3. less 用来分屏显示文件的内容。允许用户向前或向后浏览文件。
  4. head 在屏幕上显示指定文件的开头若干行。显示行数的默认值是10。
  5. tail 命令在屏幕上显示指定文件的末尾10行
  6. touch 可以修改指定文件的时间标签或者创建一个空文件
  7. file 对每个参数FILE进行检查，并予以分类。

匹配、排序及显示指定内容的命令

  • grep 在文本文件中查找指定模式的词或短语，并在标准输出上显示包括给定字符串模式的所有行。
    1. grep 一次只能搜索一个指定的模式;
    2. egrep 等同于grep-E,可以使用扩展的字符串模式进行搜索;
    3. fgrep 等同于grep -F,是快速搜索命令,它检索固定字符串,但不识别正则表达式。

比较文件内容的命令

  • comm 对两个已排序文件进行逐行比较。
    1. -1　　 不输出仅在文件1中出现的行。
    2. -2　 不输出仅在文件2中出现的行。
    3. -3　 不输出在两个文件中都存在的行。
    4. –help　 显示帮助信息并且退出。
  • diff 逐行比较两个文件，列出它们的不同之处，并且告诉用户，为了使两个文件一致，需要修改它们的哪些行。

复制、删除和移动文件的命令

  • cp 复制。 例如：cp –f file1.c myfile.c 复制文件1到我的文件中。
  • rm 删除。
    1. -f,–force　　　 忽略不存在的文件,并且不给出提示信息。
    2. -r,-R,–recursive 递归地删除指定目录及其下属的各级子目录和相应的文件。
    3. -i　 交互式地删除文件。
  • mv 用来对文件或目录重新命名，或者将文件从一个目录移到另一个目录中。
    • $ mv ex3 new1（将文件ex3改名为new1）
    • $ mv /home/mengqc/* . (将/home/mengqc/*目录下的所有文件移到当前目录)
    1. -i,–interactive　　交互式操作。
    2. -f　与-i相反,它禁止交互式操作。在覆盖已有的目标文件时,不给任何提示。
    3. -t,–target-directory=DIRECTORY 把所有的源文件或目录都移到 directory 所指示的目标中。

文件内容统计命令

  • wc 统计指定文件的字节数、字数、行数，并将统计结果显示出来。
    1. -c,–bytes　统计字节数。
    2. -l,–lines　统计行数。
    3. -m,–chars　统计字符数。
    4. -w,–words　统计字数。

目录及其操作(第三章)

目录概念

  • Linux树型
    1. 起文件控制块作用的结构称作I节点（即Inode）。在I节点中存放该文件的控制管理信息。每个文件有唯一的I节点。
    2. 文件控制块的有序集合称为文件目录。文件控制块就是其中的目录项。完全由目录项构成的文件称为目录文件。
    3. 子目录是挂靠在另一个目录中的目录。包含子目录的目录称作父目录。除了root目录以外，所有的目录都是子目录，并且有它们的父目录。root目录就作为自己的父目录。
  • 根目录
    1. bin 包含二进制文件的可执行程序。
    2. sbin 存放用于管理系统的命令。
    3. boot 用于存放引导系统时使用的各种文件。
    4. etc 非常重要，它包含许多Linux系统配置文件（如密码文件/etc/passwd、/etc/profile等）。
    5. root 是超级用户的目录。
    6. dev 包含标示设备的特别文件。
    7. home 是用户起始目录的基础目录。通常，用户的主目录就保存在该目录中。
    8. lib 保存程序运行时使用的共享库。
    9. lost+found 存放系统非正常关闭时正在处理的文件，以便下次系统启动时予以恢复。
    10. mnt 存放安装文件系统的安装点。
    11. proc 实际上是一个虚拟文件系统。
    12. tmp 用于存放程序运行时生成的临时文件。
    13. usr 包含了多个子目录，其中保存系统上一些最重要的程序，可供所有用户共享。
    14. var 包括系统正常运行时要改变的数据。
  • 工作目录和主目录
    1. 工作目录
    2. 用户目录，主目录往往位于/home目录之下

注：用户主目录包含子目录、数据文件，以及用于注册环境的配置文件。

  • 绝对路径
    • 略。
  • 相对路径
    • 略。

用户及文件存取权限

  • 用户类型
    1. 文件主 文件所有者，并赋予唯一的注册名。
    2. 同组用户 通常，组中包含了有相同需求的用户。文件用户组关系的标识是GID。
    3. 其他用户
    4. 超级用户
  • 用户存取权限
    1. r（读）
    2. w（写）
    3. x（可执行或查找）
  • 文件存取权限
    1. r（读）
    2. w（写）
    3. x（执行）
  • 目录存取权限
    1. r（读）
    2. w（写）
    3. x（查找）

创建和删除目录操作命令

  • mkdir 创建目录
  • rmdir 删除目录
    1. -p,–parents　递归删除目录dirname,当子目录删除后其父目录为空时,也一同被删除。如果有非空的目录,则该目录保留下来。
    2. -v,–verbose　对每个被处理的目录输出相关诊断信息。

改变工作目录和显示目录内容

  • cd 改变工作目录。
  • pwd 显示出当前工作目录的绝对路径。
  • ls 列出指定目录的内容。
    1. -a,–all　　　　显示指定目录下所有子目录和文件,包括以“.”开头的隐藏文件。
    2. -A,–almost-all　显示指定目录下所有子目录和文件,包括以“.”开头的隐藏文件,但是不列出“.”和“…”目录项。
    3. -b,–escape　当文件名中包含不可显示的字符时,则用\ddd(三位八进制数)形式显示该字符。
    4. -c　按文件的修改时间排序。
    5. -C　分成多列显示各项。
    6. -d　如果参数是目录,则只显示它的名字(不显示其内容)。往往与-l选项一起使用,以得到目录的详细信息。
    7. -F,–classify　在列出的文件名后面加上不同的符号,以区分不同类型的文件。可以附加的符号有:
      • /　表示目录
      • ∗　表示可执行文件
      • @　表示符号链接文件
      • |　表示管道文件
      • =　表示socket文件
    8. -i,–inode　　　在输出的第一列显示文件的I节点号。
    9. -l 以长格式显示文件的详细信息。输出的信息分成多列，它们依次是：文件类型与权限 链接数 文件主 文件组 文件大小 建立或最近修改的时间 文件名
      • -：普通文件
      • l 符号链接文件
      • d 目录
      • s 套接字文件（socket）
      • b 块设备文件
      • p 命名管道文件（pipe）
      • c 字符设备文件

链接文件的命令

Linux具有为一个文件起多个名字的功能，称为链接。

  • 硬链接
    1. 不能对目录文件做硬链接。
    2. 不能在不同的文件系统之间做硬链接。
  • 符号链接（软链接）
    1. 是将一个路径名链接到一个文件。事实上，只是一个文本文件,其中包含它提供链接的另一个文件的路径名。
    2. 符号链接确实是一个新文件，它具有不同的I节点号。
    3. 可以对目录文件做符号链接，也可以在不同文件系统之间做符号链接。
    4. 删除源文件或目录，只删除了数据，不会删除链接。
    5. 在目录长列表中，符号链接作为一种特殊的文件类型显示出来，其第一个字母是l。
    6. 符号链接的大小是其链接文件的路径名中的字节数。
    7. 当用ls -l 命令列出文件时，可以看到符号链接名后有一个箭头指向源文件或目录
  • ln 用来创建链接
    1. -f,–force　　　删除已有的目的地文件。
    2. -i,–interactive　提示是否删除目的地文件。
    3. -s,–symbolic　建立符号链接,而不是硬链接。
    4. -v,–verbose　显示每个被链接文件的名字。

改变文件或目录存取权限

  • chmod 只有文件主或超级用户root才有权用chmod命令改变文件或目录的存取权限。
    1. -c,–changes　与-v选项类似,但是仅当做了改变才报告结果。
    2. -v,–verbose　详细列出该命令对每个处理文件所做的工作。
    3. -R,–recursive　递归地修改文件和目录的权限。
    • 举例：$ chmod a+x ex1 （将文件ex1的权限改为所有用户都有执行权限）
    • 操作对象who可以是下述字母中的任一个或者它们的组合：
      1. u（user ）
      2. g（group）
      3. o（others）
      4. a（all ，系统默认值）
    • 操作符号可以是：
      1. ‘+‘ 添加
      2. ’-‘ 取消
      3. ’=‘ 赋予
    • mode所表示的权限可用下述字母的任意组合：
      1. r
      2. w
      3. x
      4. 组合等
  • 以绝对方式改变权限
    • chmod 0664 ex1 （使文件ex1的文件主和同组用户具有读、写权限，而其他用户只可读）
    • chmod 777 ex1（全部）

改变用户组和文件主

  1. chgrp 用来改变文件或目录所属的用户组。
  2. chown 改变某个文件或目录的所有者和/或所属的组。

联机帮助命令

  • man
    1. name:表示命令的名称和用法。
    2. synopsis:显示命令的语法格式,其中列出所有可供使用的选项及参数。方括号中的内容是可选的。
    3. description:描述命令的一般用法和每个选项的功能。
    4. author:说明编写这个程序的作者。
    5. bugs:如果用户发现该程序有问题,可以向指定机构报告。
    6. copyright:自由软件版权声明。
    7. seealso:说明命令的其他方面或对命令的其他解释。

进程及其管理（第四章）

程序并发执行的特征

  1. 失去封闭性。
  2. 程序与计算不再一一对应。
  3. 并发程序在执行期间相互制约。

进程概念

  1. 进程定义（task）：程序在并发环境中的执行过程。
    • “进程”是操作系统的最基本、最重要的概念之一。
    • 进程最根本的属性是动态性和并发性。
  2. 进程的基本特征
    • 动态性（重要）
    • 并发性（重要）
    • 调度性
    • 异步性
    • 结构性
  3. ps 是查看进程状态的最常用的命令,它可以提供关于进程的许多信息

进程管理

  • kill 通过向进程发送指定的信号用来终止一个进程的运行。

其他常用命令

磁盘使用情况统计

  • df 可以报告文件系统中未用磁盘空间的情况。按照默认，磁盘盘块以1k字节为单位。
  • du 统计出每个目录占用磁盘空间的情况。该命令可以逐级进入每一个子目录并显示该目录的磁盘利用情况。

文件压缩和解压缩

  • gzip 用来对文件进行压缩和解压缩。
    1. -c,–stdout,–to-stdout　　　　将输出写到标准输出上,并保留原有文件。
    2. -d,–decompress,–uncompress　将被压缩的文件进行解压缩。
    3. -l,–list　　对每个压缩文件,列出以下字段:
      • compressedsize:压缩文件的大小
      • uncompressedsize:未压缩文件的大小
      • ratio:压缩比(未知时为0.0%)
      • uncompressed_name:未压缩文件的名字
    4. -r　　　递归地查找指定目录并压缩其中的所有文件或者是解压缩。
    5. -t　　　测试,即检查压缩文件的完整性。
    6. -v　　　对每个压缩文件和解压缩文件,显示其文件名和压缩比。
    7. -num 用指定的数字num 调整压缩速度,其中-1或–fast表示最快的压缩方法(低压缩比),-9或–best表示最慢的压缩方法(高压缩比)。系统默认值为-6。
  • unzip 对ZIP格式的压缩文件进行解压缩。这种格式的压缩文件带有后缀.zip。

文本编辑（第五章）

本章内容提要

  1. 进入和退出vi
  2. vi的工作方式
  3. 文本输入命令
  4. 光标移动命令
  5. 文本修改命令
  6. 字符串检索

编辑器

  1. 行编辑器。(如ed,ex,edit)
  2. 屏幕编辑器。(如vi)

vi是visual interface的简称。在Linux系统中,还提供了vim(Vi IMproved)编辑器,它是vi的增强版本,与vi向上兼容。

进入vi

  • vi xxx.xx 进入编辑

退出vi

  • :wq 把编辑缓冲区的内容写到你编辑的文件中，退出编辑器，回到 shell下。
  • :ZZ 仅当作过修改时才将缓冲区内容写到文件上。
  • :x 同上。
  • :q! 强行退出vi。感叹号（! )告诉vi，无条件退出，丢弃缓冲区内容。

vi的工作方式

  1. 命令方式。 输入命令vi进入编辑器时，就处于vi的命令方式。
  2. 插入方式。 输入vi的插入命令( i )、附加命令( a )、打开命令(o )、替换命令( s )、修改命令( c )或取代命令( r )可以从命令方式进入到插入方式。
  3. Ex转义方式 在命令方式下输入一个冒号（:）就进入 ex转义方式。

文本输入命令

插入命令

  • i 插在光标位置之前。
  • I 在光标所在行的行首插入新增文本

附加命令

  • a 在该命令之后输入的字符都插到光标之后。
  • A 在光标所在行的行尾添加文本。

打开命令

  • o 在光标所在行的下面新开辟一行，随后输入的文本就插入在这一行上。
  • O 在光标所在行的上面新开辟一行,随后输入的文本就插入在这一行上。

插入方式下光标移动

  • 键盘四个方向键，利用它们可以在插入方式下移动光标。
  • 利用（退格键）可将光标从当前行上回退一个字符，并且删除光标之前的一个字符。

光标移动命令

  1. 向左移动一个字符

使用命令 h (小写字母)、、左向键将光标向左移动一个字符。

  1. 移到下一行

使用 j、和下向键分别将光标向下移一行，但所在列不变。使用命令 +、将光标移到下一行的开头。

  1. 移到上一行

使用命令 －、k（小写字母）、P 、上向键将光标上移一行。

  1. 向右移动一个字符

使用 l（小写字母）、、右向键将光标向右移动一个字符。

  1. 移至行首

使用命令（键）^ 或0（数字0）将光标移到当前行的开头。例如：
int main ( )
{
printf ( "Hello ! \ n " ); （光标位于字母e上）
}
输入^命令，光标移至字母p；再输入0命令，光标左移至该行的第一列（空格）上。

  1. 移至行尾

可以使用命令（键）＄将光标移至当前行的行尾，停在最后一个字符上。

  1. 移至指定行

可以使用命令（键）［行号］G将光标移至由行号所指定的行的开头。

  1. 移至指定列

可以使用命令（键）［列号］| 将光标移至当前行指定的列上。

  • 如果在相应命令的前面加上一个数字n ，那么，就把光标相应移动n个字符。例如：
    1. 6l，则向右移6个字符（但不能跨行）；
    2. 4h ，则向左移动4个字符 （但不能跨行）；
    3. 3＋，则向下移3行，光标位于行首；
    4. 6k ，则光标上移6行，列数不变。

文本修改命令

文本删除

  1. 删除字符
    • 命令x（小写字母）删除光标所在的字符。
    • 命令X（大写字母）删除光标前面的那个字符。
  2. 删除文本对象
    • 命令dd 删除光标所在的整行。
    • 命令D 从光标位置开始删除到行尾。
    • 组合命令d+<光标移动命令> 删除从光标位置开始至光标移动命令限定的位置之间的所有字符。例如：
      1. d0 从光标位置（不包括光标位）删至行首。
      2. d3l 从光标位置（包括光标位）向右删3个字符。
      3. d$ 从光标位置（包括光标位）删至行尾。与D相同。
      4. d5G 将光标所在行至第5行都删除。

文本复原

  1. 复原命令
    • 命令u 取消前面刚执行的插入或删除命令的效果，恢复到此前的情况。
    • 命令U 总是把当前行恢复成它被编辑之前的状态。
  2. 重复命令
    • ·（圆点）在命令方式下，重复执行前面最近一次插入命令或删除命令。
  3. 修改命令
    • 命令c 紧随光标移动命令（用来限定删除文本的范围），之后是新输入的文本，最后单击 < Esc >键。
    • 命令C 紧接新输入的文本，最后单击< Esc >键。它等价于：c$。
    • 命令cc 删除光标所在行整行（不是行的一部分），用随后输入字符串替代。其余作用与C命令相同。

C和cc前面还可以加上数字——表示要修改的行数

  1. 取代命令
    • 命令r 用随后输入的单个字符取代光标所在的字符。
    • 命令R 用随后输入的文本取代光标所在字符及其右面的若干字符，每输入一个字符就替代原有的一个字符。
  2. 替换命令
    • 命令s（小写）用随后输入的的正文替换光标所在的字符。
    • 命令S（大写）用新输入的正文替换整个当前一行。

字符串检索

  • 向前检索命令的基本格式是：/模式〈Enter〉例如： /sTudent 〈Enter〉
  • 向后检索命令的基本格式是：？模式 〈Enter〉例如：?flout 〈Enter〉
