第一章 移动操作系统概论

1.操作系统原理和概念

  • 操作系统 用于管理、控制计算机软硬件的软件。为用户提供高效、可靠和安全的服务。

2.操作系统历史

  1. 第一代计算机（1945-1955）：电子管时代，无操作系统。
  2. 第二代计算机（1955-1965）：晶体管时代，采用批处理系统。
  3. 第三代计算机（1965-1980）：集成电路时代，多道程序设计。
  4. 第四代计算机（1980-至今）：大规模和超大规模集成电路，分时系统

3.操作系统的分类

  • 批处理操作系统（现代操作系统的基础，现代的操作系统中很多功能依然采用批处理设计。）
    • 单道批处理程序 每一时刻只能有一个作业在内存中
    • 多道批处理程序 能并发执行多个作业
    • 优点：缩短作业间交接时间，减少处理器的空闲等待时间，提高了系统吞吐量。
    • 缺点：执行过程不能交互，不能控件计算机，作业周期长。
  • 分时操作系统（采用时间片段轮流方式，能为多个终端用户服务，并提供交互能力。）
    • 优点：交互性，用户独立性，多用户同时性
  • 实时操作系统（实时控制，实时信息处理的操作系统。）
    • 实时控制
    • 实时信息处理
  • 个人计算机操作系统
    • 交互式
    • 单用户
  • 网络操作系统（将多个独立的、分散的和具有自治功能的计算机连接起来）
  • 分布式操作系统（更高级的“网络操作系统”）
  • 嵌入式操作系统（嵌入式设备使用的操作系统）
    • 特点：
      • 系统内核小
      • 专用性强
      • 系统精简
      • 高实用性
      • 多任务操作系统
      • 需要开发工具和环境
    • 移动操作系统是嵌入式操作系统的一个分支。

并发：一个人吃三碗饭。（同一时间间隔内）

    • 指纹识别传感器
    • 环境光传感器
  • 摄像头
  • 网络通信

第三章 第一个Android应用程序

Java包名规范

1. 一般都是小写字母
2. 首字母不能是数字
3. 不能有空格

res资源目录

1. drawable 存放应用程序需要的图片文件。
2. layout 布局文件
3. mipmap 存放资源文件，加快资源渲染速度，提高照片质量，减少CPU压力。
4. value 参数值目录，存放应用所需要显示的文字和数据。

androidManifest.xml文件

1. 声明应用的Java源代码包名
2. 描述应用中的组件
3. 声明应用必须具备的权限（例如访问新浪微博的权限）
4. 声明应用所需的最低Android API级别
5. 声明应用的安全控制和测试等信息

进程优先级（从高到低）

1. 前台进程、
2. 可见进程、
3. 服务进程、
4. 后台进程
5. 空进程

2.Android事件处理过程三个概念

  1. 事件源 事件发生场所（各个控件）
  2. 事件 用户在界面上的操作描述，可封装为一个类的形式出现，与JavaSE图形界面不一样的是并不是所有的事件都被封装成为一个类
  3. 事件处理者 接收事件对象并对其进行处理，事件处理一般是实现某些特定接口类创建的对象

3.基本宽高单位

（1）px：像素，屏幕中可显示最小元素单位。设置控件大小，分辨率不同的手机上控件显示出来的大小不同
（2）pt：磅数，字体单位。与px较相似，不同分辨率的手机，用pt作字体单位显示的大小不一样
（3）dp：密度，无关像素（dip），作为控件与布局宽高单位。无论屏幕分辨率如何，能显示相同大小。
（4）sp：可伸缩像素，设置字体大小。与dp相同设计理念。

第九章 活动

活动

  • Activity是一个界面管理员

    • onKeyDown(int keyCode,KeyEvent event)：按下事件
    • onKeyUp(int keyCode,KeyEvent event)：松开事件
    • onTouchEvent(MotionEvent event)：单击屏幕事件

  • Activity交互机制

    • Android针对Activity管理使用栈机制，Activity栈保存已启动，且没有终止Activity，遵循“后进先出”原则。
    • Activity直接涉及用户交互界面处理，任意时刻与用户交互界面只有一个，采用具有层次感的栈数据结构。

  • 活动三种状态

    • 运行状态
    • 暂停状态
    • 停止状态

  • 七个方法

    • onCreate() 活动初始化时调用
    • onStart() 活动从不可见变为可见，但还是暗色，不能获得焦点，不能接收用户事件
    • onResume() 活动从暗色可见变成高亮可见，可获得焦点，接收用户事件
    • onPause() 活动从运行状态到暂停状态
    • onStop() 活动从暂停状态到停止状态
    • onRestart() 处于停止状态的活动，重新回到前台，变成活动状态时调用
    • onDestroy() 活动被销毁时调用

  • 三个嵌套循环

    • 整个生命周期
    • 可见生命周期
    • 前台生命周期循环

第十四章 Android多线程开发

线程概念

  • 线程是进程一部分。一个进程内的基本调度单位–线程
  • 多线程：单个程序中可同时运行多个不同线程，执行不同任务

进程概念

  • 具有独立功能程序对某个数据集在处理机上的执行过程和分配资源基本单位。

线程模型

1、单线程模型
一个进程中只能有一个线程运行。
2、多线程模型
可同时被多个线程调用，调用者不需要任何动作（同步）确保线程安全性。

Android异步消息处理机制

  • 解决主线程和子线程之间异步通信问题。开发人员可在子线程中进行计算、耗时处理等任务，完成后将消息发送给主线程，由主线程更新UI。

第十五章 服务

什么是服务？

  • 长生命周期的后台运行的组件，不提供任何用户界面。

服务的特点

1、无法与用户直接交互
2、必须由用户或者其他程序启动
3、运行优先级比前台应用低，但比后台其他应用高

服务分类

  • 启动类型服务
  • 绑定类型服务

IntentService的优势

创建默认工作线程，用于主线程后执行传递给onStartCommand()所有意图
创建工作队列，将意图逐一传递给onHandleIntent()方法实现，不用担心多线程问题
处理完所有启动请求后自动停止服务，不需要调用stopSelf()方法
提供onBind()默认实现，返回null
提供onStartCommand()默认实现，将意图依次发送到工作队列和onHandleIntent()实现

Service本身存在问题

1、不会专门启动单独进程，与所在应用位于同一进程；
2、不会专门启动新线程，不应在service中处理耗时操作

IntentService弥补

1、创建单独worker线程处理所有intent请求
2、创建单独worker线程处理onHandleIntent()方法实现代码
3、所有请求处理完后，Intentservice会自动停止
